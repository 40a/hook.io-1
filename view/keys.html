Keys

Keys are used to manage security role access to services on hook.io

You can create new keys to allow third-party services or users to access restricted hook.io services like privates hooks, the datastore, or logs.

A full listing of security access roles is available at /roles

A common use-case would be creating a private hook service which is only accessible to clients who can provide a generated access key which has the role hook::run

Understanding Keys and Roles

When creating a new access key, you must specify a set of roles associated with that key.

These roles determine what the new access key has permission to do.

Using Keys and Roles to Access Restricted Services

Using Keys with Roles is easy!

All you have to do is supply the hook_private_key variable as a HTTP request parameter ( URL GET Query Data / POST Form Data / JSON RPC / etc ) and that HTTP client request will be granted the roles associated with that public key.

All service level events on hook.io are available as roles, allowing granular access control.
