Keys

Keys are used to manage security role access to services on hook.io

You can create new keys to allow third-party services or users to access restricted hook.io services like privates hooks, the datasource, or logging.

A full listing of security roles is available for review at /roles

Using Keys and Roles to Access Restricted Services

Using keys is easy!

All you have to do is supply the hook_public_key and hook_private_key variables as HTTP request parameters ( URL GET Query Data / POST Form Data / JSON RPC ) and you will receive admin access to the resource.

In the near future, we'll be implementing a more advanced role management system to allow for individual role access per key per resource.
