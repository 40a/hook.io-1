Hosted Microservices and Webhooks

It's important to understand that you don't have to use the {{appName}} API from an outside system

{{appName}} provides hosting for Microservices and Webhooks.

These hosted services has full integrated access to the {{appName}} API. This is very useful if you intend to run a block of code in response to HTTP requests

API Access Keys

Any authorized usage of the {{appName}} will require an API access key

Generating an API access key is simple. Just visit /keys and create a new key with the required roles you wish to use on the API

Generate new keys with granular roles per service in order to maximize your account's security.

REST HTTP API

All services on {{appName}} can be access over simple HTTP requests

Every page on {{appName}} is also an API endpoint capable of receiving data from a JSON post ( or `application/x-www-form-urlencoded`)

The content of the rendered result will be determined by the `accept` header of the incoming HTTP request to the API

We'll be releasing a nicely formatted API definition file soon. Until then, we recommend checking out the {{appName}}-sdk.

Developer SDK

The easiest way to programmatically interact with the the {{appName}} HTTP API is by using the {{appName}}-sdk client.

This SDK provides a full mapping of all {{appName}}'s API methods in an easy to use interface

If you'd like to see additional API SDK clients added, you can open an issue here.
